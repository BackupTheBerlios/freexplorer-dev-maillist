Après test il faudras surement une interface différente.
Mono supporte bien les forms du .Net 1.0 par contre celle du 2.0 il
lui manque beaucoup de fonctionnalitées pour les supporter.

Je pense que je vais retourner sous windows et séparer l'interface du
code de chargement de la config puis extraire l'interface dans un
assembly différent.
En suite je créerait un FreeXplorerConsole qui ne nécessite pas
d'interface graphique (Et qui n'est donc configurable qu'en éditant le
fichier XML) et adapterait tout ça sous linux.

Pour info les quelques choses qui manquent sur la version actuelle de
mono pour compiler le code à par les forms c'est :
 - System.IO.File.ReadAllText (
http://bugzilla.ximian.com/show_bug.cgi?id=77813 ), facile à émuler
avec un StreamReader.ReadToEnd().
 - TCPListener.Server car ce membre est privé (Je vais leur envoyer un
mail mais il faut dabord que je vérifie si il à toujour été public ou
si c'est un changement dans le 2.0.
 - La classe Registry et tout ce qui est dans Microsoft.Win32
(Logique), pour la gestion des types Mime par exemple

Evidement une fois compilé il y auras surement d'autres problèmes à
l'exécution mais je n'en suis pas là :-D
